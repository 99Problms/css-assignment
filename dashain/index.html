Header tag

The <header> element represents a container for introductory content or a set of navigational links.

A <header> element typically contains:

  • One or more heading elements (<h1> - <h6>)
  • Logo or icon
  • Authorship information

Footer Tag

The <footer> tag defines a footer for a document or section.

A <footer> element typically contains:

  • Authorship information
  • Copyright information
  • Contact information
  • Sitemap
  • Back to top links
  • Related documents

Article Tag

The <article> tag specifies independent, self-contained content.

An article should make sense on its own and it should be possible to distribute it independently from the rest of the site.

Potential sources for the <article> element:

  • Forum post
  • Blog post
  • News story

Section Tag

The <section> tag in HTML5 defines a generic standalone section of a document. It is a semantic element used for grouping together related content that forms a distinct, thematic grouping within a larger document or application.

CSS (Cascading Style Sheets) is a language used to style and layout HTML elements on a webpage. While HTML structures the content, CSS controls how it looks and feels—including colors, fonts, spacing, positioning, and responsiveness.

Why CSS is used:

  1. Separation of content and design: Keeps HTML clean by separating structure from styling.
  2. Consistent styling: Apply the same styles to multiple pages or elements easily.
  3. Responsive design: Adjust layout for different devices (desktop, tablet, mobile).
  4. Enhanced user experience: Makes web pages visually appealing and easier to navigate.
  5. Efficiency: Changing one CSS rule can update styles across an entire site.

Type of CSS Description Example
Inline CSS Styles are applied directly to an HTML element using the style attribute. <p style="color: blue; font-size: 16px;">Hello</p>
Internal CSS Styles are written inside a <style> tag within the <head> section of the HTML document. <style>

</style>
External CSS Styles are written in a separate .css file and linked to the HTML using the <link> tag. <link rel="stylesheet" href="style.css">

Points Flexbox Grid
Definition One-dimensional layout system for arranging items in a **row or column**. Two-dimensional layout system for arranging items in **rows and columns**.
Main Axis Works along **one main axis** (horizontal or vertical). Works along **both axes** (rows and columns) simultaneously.
Use Case Best for **aligning items** in a single line (e.g., navigation bar). Best for **page layouts** or complex designs (e.g., web page sections).
Control Control over **content flow and alignment** within a single direction. Control over **precise placement** of elements in a structured grid.
Syntax Example display: flex; display: grid;
Alignment Uses justify-content and align-items for alignment along one axis. Uses justify-items, align-items, and grid-template for both axes.
Item Placement Order depends on **source (HTML) order**. Items can be placed **anywhere** in the grid using row and column numbers.
Responsiveness Good for resizing items in a line. Better for **overall page layout responsiveness**.
Example Use Menus, toolbars, card lists. Full webpage layouts, dashboards, galleries.

Aspect Mobile-First Design Desktop-First Design
Approach Design starts from smaller screens (mobile) and expands to larger ones. Design starts from larger screens (desktop) and shrinks down to smaller devices.
